# 特殊处理

## 概述

某些特殊情况需要特殊处理,skill-optimizer必须遵循以下规则。

## 特殊SKILL处理规则

### defect-analyzer特殊处理

- **绝对禁止**: 绝不修改defect-analyzer的references目录
- **原因**: references目录是defect-analyzer的知识库,由defect-analyzer自身维护(由defect-document-generator自动生成)
- **处理方式**: 如果需要优化defect-analyzer,只优化SKILL.md,不触碰references目录

### 其他SKILL的处理

对于除defect-analyzer之外的所有SKILL,可以优化所有文件:
- SKILL.md
- scripts/
- references/
- assets/

## 主动持续迭代中的特殊处理

在主动持续迭代时,对于不同的SKILL采取不同的处理策略:

### 1. 分析SKILL的类型

根据SKILL的特性和功能,确定迭代策略:

- **工具类SKILL**: 如APK安装、打包构建等工具类SKILL
  - 检查工具的使用模式
  - 优化工具的易用性
  - 改进工具的效率

- **检查类SKILL**: 如代码规范检查、问题分析等检查类SKILL
  - 检查检查的覆盖率
  - 优化检查的准确性
  - 改进检查的性能

- **生成类SKILL**: 如需求文档生成、代码生成等生成类SKILL
  - 检查生成的质量
  - 优化生成的逻辑
  - 改进生成的效果

### 2. 根据SKILL类型调整迭代策略

- **工具类SKILL**:
  - 关注用户体验
  - 关注操作流程
  - 关注错误提示

- **检查类SKILL**:
  - 关注检查项完整性
  - 关注检查逻辑正确性
  - 关注检查效率

- **生成类SKILL**:
  - 关注生成质量
  - 关注生成逻辑
  - 关注用户满意度

### 3. 特殊情况处理

对于以下特殊情况,需要特别注意:

- **用户反馈不满意**: 立即分析原因,优化相关SKILL
- **功能缺失**: 立即补充功能,更新文档
- **逻辑错误**: 立即修正逻辑,更新文档
- **性能问题**: 立即优化性能,更新文档
- **兼容性问题**: 立即解决兼容,更新文档

## 特殊处理流程

1. **识别特殊情况**
2. **分析特殊原因**
3. **设计特殊方案**
4. **执行特殊处理**
5. **验证特殊效果**
6. **更新相关文档**
