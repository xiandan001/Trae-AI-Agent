# 最佳实践

## 概述

本文档总结了skill-optimizer在执行优化时应该遵循的最佳实践。

## 优化前准备

### 1. 充分理解问题
- 仔细阅读用户的问题描述
- 理解问题的上下文
- 识别问题的核心需求

### 2. 充分理解SKILL
- 仔细阅读目标SKILL的SKILL.md
- 理解SKILL的功能范围
- 理解SKILL的工作流程

### 3. 分析问题与SKILL的关系
- 确定问题是否由SKILL导致
- 评估问题的影响范围
- 确定优化的必要性

## 优化过程

### 1. 遵循工作流程
- 严格按照工作流程执行
- 不跳过任何步骤
- 确保每个步骤都完成

### 2. 详细记录过程
- 记录分析的详细过程
- 记录判断的依据
- 记录优化的理由

### 3. 保持透明度
- 向用户说明分析过程
- 向用户说明判断依据
- 向用户说明优化内容

### 4. 保持客观性
- 基于事实做判断
- 不主观臆断
- 不预设结论

## 优化实施

### 1. 精确定位修改位置
- 使用搜索功能精确定位
- 验证位置的正确性
- 避免修改错误位置

### 2. 遵循修改原则
- 保持SKILL的结构
- 遵循文档规范
- 确保修改的准确性

### 3. 验证修改效果
- 检查修改是否解决问题
- 检查修改是否引入新问题
- 确认修改符合预期

### 4. 记录修改日志
- 使用规范的格式记录
- 确保记录的完整性
- 确保记录的准确性

## 验证与反馈

### 1. 充分验证
- 测试所有相关场景
- 验证修改的有效性
- 确认没有副作用

### 2. 清晰反馈
- 说明优化的内容
- 说明优化的效果
- 说明可能的影响

### 3. 记录日志
- 记录到change-log
- 使用规范的命名
- 确保信息完整

## 自我优化

### 1. 持续学习
- 从每次优化中学习
- 从错误中学习
- 从用户反馈中学习

### 2. 持续改进
- 改进判断策略
- 改进优化方法
- 改进工作流程

### 3. 主动迭代
- 主动发现问题
- 主动提出优化
- 持续提升质量

## 常见错误

### 1. 过度依赖关键词
- **错误**: 只根据关键词判断,忽略上下文
- **后果**: 错误判断相关性
- **改进**: 加强上下文分析的权重

### 2. 只记录不修改
- **错误**: 只生成change-log,不修改SKILL
- **后果**: 问题未解决
- **改进**: 必须实际修改SKILL文件

### 3. 路径错误
- **错误**: change-log路径错误
- **后果**: 日志无法追溯
- **改进**: 使用正确的SKILL名称作为目录名

### 4. 时间格式错误
- **错误**: 使用不准确的时分秒
- **后果**: 文件名冲突
- **改进**: 使用问题描述英文缩写+年月+月内序号

## 总结

遵循最佳实践可以:
- 提高优化的准确性
- 提高优化的效率
- 提高用户满意度
- 减少错误和重复工作
