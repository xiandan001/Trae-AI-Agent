# 问题诊断流程

## 概述

问题诊断是skill-optimizer的第二阶段,在判断为"强相关"后进入。通过深入分析问题,定位需要优化的具体位置和内容。

## 诊断步骤

### 1. 读取目标SKILL

- 读取被优化的SKILL的SKILL.md文件
- 分析SKILL的结构和组织方式
- 理解SKILL的功能范围和工作流程

### 2. 定位问题类型

根据相关性判断阶段的问题分析,确定问题类型:

- **内容缺失**: SKILL缺少某些必要的检查项、说明、示例等
- **逻辑错误**: SKILL的逻辑存在错误或矛盾
- **表述不清**: SKILL的某些表述模糊,容易产生误解
- **执行问题**: SKILL的执行流程有问题,导致执行结果不符合预期
- **架构问题**: SKILL的整体架构设计存在缺陷

### 3. 问题定位

根据问题类型,精确定位问题所在的位置:

#### 内容缺失

- 分析用户反馈的功能需求
- 对比SKILL中已有的内容
- 确定缺失的内容项
- 定位缺失内容的章节和行号

#### 逻辑错误

- 分析用户的反馈和SKILL的逻辑
- 找出逻辑矛盾的点
- 确定需要修改的逻辑
- 定位逻辑错误的章节和行号

#### 表述不清

- 分析用户产生误解的原因
- 找出表述模糊的地方
- 确定需要澄清的表述
- 定位表述不清的章节和行号

#### 执行问题

- 分析执行流程的问题点
- 找出导致执行错误的原因
- 确定需要调整的流程
- 定位执行问题的章节和行号

#### 架构问题

- 分析SKILL的整体架构
- 找出架构设计的缺陷
- 确定需要重构的部分
- 定位架构问题的范围

### 4. 问题详情记录

在定位问题后,详细记录:

- **问题类型**: 具体的问题类型
- **问题位置**:
  - 文件路径
  - 章节名称
  - 行号范围
- **问题详情**: 问题的具体描述
- **影响范围**: 问题影响的范围
- **严重程度**: 问题的严重程度(轻微、一般、严重)

## 诊断输出

问题诊断阶段的最终输出:

1. 问题类型
2. 问题位置(文件、章节、行号)
3. 问题详情
4. 影响范围
5. 严重程度

这些信息将传递给优化方案设计阶段,用于设计针对性的优化方案。
