# 错误处理

## 概述

在优化过程中可能出现各种错误,skill-optimizer需要能够正确处理这些错误。

## 相关性判断错误

当skill-optimizer判断问题与SKILL不相关,但实际上问题确实由SKILL导致时,即为相关性判断错误。

**错误处理流程**:
1. 用户反馈"不是这样的"、"这样分析不对"等问题
2. 重新判断相关性
3. 创建ClAUDE-ERROR.md
4. 优化相关SKILL
5. 自我优化

**错误记录**: 参见[workflow-relevance-judgment.md](./workflow-relevance-judgment.md)中的"错误记录机制"部分

## 问题诊断错误

在问题诊断阶段,可能因为以下原因出错:
- SKILL理解不够深入
- 问题定位不准确
- 问题类型判断错误

**处理流程**:
1. 重新分析用户反馈
2. 重新读取SKILL内容
3. 重新定位问题
4. 重新设计优化方案

## 优化方案错误

在优化方案设计阶段,可能因为以下原因出错:
- 优化目标不明确
- 优化策略不合理
- 影响评估不全面

**处理流程**:
1. 重新分析问题
2. 重新确定优化目标
3. 重新设计优化方案
4. 向用户确认新方案

## 优化实施错误

在优化实施阶段,可能因为以下原因出错:
- 修改内容错误
- 修改位置错误
- 修改类型错误

**处理流程**:
1. 撤销错误的修改
2. 重新实施优化
3. 验证修改的正确性

## 验证失败

**错误处理流程**:
1. 分析验证失败的原因
2. 重新设计优化方案
3. 重新实施优化

## ClAUDE-ERROR.md的使用

**文件位置**: 
- 优先位置: 项目根目录`ClAUDE-ERROR.md`
- 备选位置: `${SKILLS_ROOT}/skill-optimizer/ClAUDE-ERROR.md`

**文件作用**:
- 记录skill-optimizer的所有判断错误
- 总结经验教训,避免重复错误
- 提供改进方向,持续优化判断策略
- 作为知识资产,供自我优化时参考

**路径说明**:
- 项目根目录: 当前工作目录或项目目录的根目录
- `${SKILLS_ROOT}`: SKILLS_ROOT环境变量指定的目录,或按优先级查找的skills目录
- 优先在项目根目录创建,如果失败则在skill-optimizer目录下创建

**使用场景**:

1. **相关性判断错误**:
   - 当判断为"不相关",但用户反馈"不是这样"时
   - 创建错误记录,分析判断过程
   - 总结为什么判断错误

2. **自我优化参考**:
   - 触发自我优化时,先读取ClAUDE-ERROR.md
   - 分析历史错误模式
   - 改进相关性判断策略

3. **知识积累**:
   - 随着时间积累,形成错误知识库
   - 帮助避免类似的判断错误
   - 提供优化方向和改进建议

**记录格式**:

```markdown
# Claude分析错误记录

---

## <problem-description>-yyyy-mm-dd

### 用户原始描述
<用户的问题描述>

### 问题上下文
<问题的上下文信息>

### 错误原因
<简短描述为什么判断错误,1-3句话>

### 修正后的执行
<简短描述修正后应该怎么执行,1-3句话>

---
```

**命名说明**:
- `<problem-description>`: 问题描述(非常简洁,使用英文,如: static-context-leak, wrong-patch-label等)
- `yyyy-mm-dd`: 年-月-日(如: 2025-02-03)
