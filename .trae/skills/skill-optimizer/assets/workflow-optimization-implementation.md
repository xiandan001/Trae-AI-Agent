# 优化实施流程

## 概述

优化实施是skill-optimizer的第四阶段,根据优化方案设计阶段的输出,执行具体的修改操作。

## 实施步骤

### 1. SKILL.md修改

**修改原则**:
- 严格遵循优化方案设计
- 保持SKILL的整体结构
- 确保修改的准确性
- 遵循代码规范

**修改流程**:
1. 读取SKILL.md文件
2. 定位需要修改的内容
3. **执行修改(新增、修改、删除)** - **这是必须的步骤,不能跳过**
4. 检查修改后的内容是否正确
5. 保存文件

**重要说明**:
- **必须实际修改目标SKILL的文件**,change-log只是记录,不能替代实际修改
- 如果判断为"强相关",必须完成整个优化流程,包括实际修改文件
- change-log是优化完成后的记录,不是优化本身
- **绝对禁止**只生成change-log而不实际修改目标SKILL

**修改类型**:

#### 新增内容

- 添加缺失的章节或小节
- 补充必要的说明
- 添加示例代码或配置

#### 修改内容

- 修正错误的内容
- 澄清模糊的表述
- 优化执行流程

#### 删除内容

- 删除过时的内容
- 移除冗余的信息
- 清理无效的章节

### 2. scripts/修改

**修改流程**:
1. 读取需要修改的脚本文件
2. 定位需要修改的代码
3. 执行修改
4. 测试修改后的脚本
5. 保存文件

**修改原则**:
- 保持脚本的功能性
- 遵循编码规范
- 确保脚本的兼容性
- 添加必要的注释

### 3. references/修改

**修改流程**:
1. 读取需要修改的参考文件
2. 定位需要修改的内容
3. 执行修改
4. 验证修改的正确性
5. 保存文件

**修改原则**:
- 保持参考文件的结构
- 确保信息的准确性
- 遵循文档规范
- 更新相关引用

## 修改验证

在每次修改后,进行验证:

- 检查修改后的内容是否正确
- 验证修改是否解决了问题
- 确认修改是否引入新问题
- 检查修改是否符合规范

## 实施输出

优化实施阶段的最终输出:

1. 修改的文件列表
2. 每个文件的修改类型(新增、修改、删除)
3. 修改位置(章节、行号)
4. 修改前后的对比(可选)

这些信息将传递给验证与反馈阶段,用于验证优化效果。
