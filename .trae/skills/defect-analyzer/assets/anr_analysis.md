# ANR（Application Not Responding）通用处理流程 - EDLA环境

## 1. ANR类型识别

### 1.1 常见ANR类型
- **主线程阻塞**：主线程执行耗时操作导致无响应
- **死锁**：多线程间相互等待导致死锁
- **服务超时**：Service执行时间超过超时限制
- **BroadcastReceiver超时**：BroadcastReceiver执行时间超过限制
- **ContentProvider超时**：ContentProvider操作超时

### 1.2 ANR触发条件
- 输入事件超时（5秒）：触摸事件、按键事件等
- Service启动超时（20秒）：前台服务（10秒）、后台服务
- BroadcastReceiver超时（10秒）：前台广播
- ContentProvider超时（10秒）：内容提供者操作

## 2. 日志收集

### 2.1 必需日志
1. **anr traces文件**：/data/anr/traces.txt
   - 包含所有线程堆栈信息
   - 定位主线程阻塞位置
2. **应用日志**：应用崩溃前的日志输出
3. **系统日志**：kernel、system_server等相关日志

### 2.2 可选日志
1. **CPU使用情况**：top、ps等命令输出
2. **内存使用情况**：dumpsys meminfo
3. **进程信息**：ps、kill相关进程信息

### 2.4 日志获取方法
```bash
# 获取ANR traces文件
adb pull /data/anr/traces.txt ./

# 获取应用日志
adb logcat -v time > app_log.txt

# 获取系统日志
adb logcat -b kernel -v time > kernel_log.txt
adb logcat -b system -v time > system_log.txt

# 获取定时任务日志
adb logcat | grep -i schedule > schedule_log.txt
```

## 3. 分析步骤

### 3.1 主线程分析
1. **定位主线程**：查找"main"线程堆栈
2. **分析阻塞位置**：查看主线程当前执行的方法
3. **识别耗时操作**：查找网络请求、数据库操作、文件IO等

### 3.2 线程状态分析
1. **线程状态**：查看各线程的状态（BLOCKED、WAITING、RUNNABLE）
2. **锁等待**：分析是否存在死锁或锁竞争
3. **线程关系**：分析线程间的依赖关系
4. **EDLA服务线程**：
   - 检查定时任务线程状态

### 3.3 常见问题识别
1. **主线程执行耗时操作**
   - 网络请求
   - 数据库操作
   - 文件读写
   - 复杂计算

2. **死锁场景**
   - 多个线程相互等待锁
   - 锁顺序不一致

3. **内存泄漏**
   - 大对象占用内存
   - 静态变量引用

## 4. 解决方案

### 4.1 主线程阻塞解决
1. **异步处理**：使用AsyncTask、HandlerThread、RxJava等
2. **线程池**：使用ExecutorService、ThreadPoolExecutor
3. **协程**：使用Kotlin协程进行异步操作

### 4.2 死锁解决
1. **统一锁顺序**：确保所有线程以相同顺序获取锁
2. **超时机制**：设置锁获取超时时间
3. **锁分离**：减少锁的粒度，降低竞争

### 4.3 内存优化
1. **及时释放**：及时释放大对象引用
2. **使用弱引用**：WeakReference避免内存泄漏
3. **对象池**：复用对象，减少创建销毁

## 5. 预防措施

### 5.1 代码规范
1. **主线程只做UI操作**：禁止在主线程执行耗时操作

## 6. 常见ANR场景及处理

### 6.1 网络请求ANR
**问题**：主线程执行网络请求
**解决**：使用OkHttp + RxJava或协程进行异步请求

### 6.2 数据库ANR
**问题**：主线程执行数据库查询
**解决**：使用Room数据库的异步查询或后台线程操作

### 6.3 文件IO ANR
**问题**：主线程读写文件
**解决**：使用后台线程处理文件操作

### 6.4 布局加载ANR
**问题**：布局过于复杂导致加载缓慢
**解决**：优化布局层级、使用ViewStub延迟加载

### 6.5 应用启动限制ANR
**问题**：检查应用启动限制时ANR
**原因**：主线程执行复杂的权限和策略检查
**解决**：
- 异步执行权限和策略检查
- 缓存应用启动限制信息
- 优化检查逻辑

### 6.6 定时任务ANR
**问题**：执行定时任务时ANR
**原因**：任务执行时间过长或任务数量过多
**解决**：
- 控制单个任务执行时间
- 使用工作队列管理任务
- 优化任务执行逻辑

## 8. 案例分析

### 8.1 案例一：主线程网络请求
**现象**：应用启动时ANR
**原因**：Application.onCreate中执行网络请求
**解决**：将网络请求移至后台线程

### 8.2 案例二：死锁
**现象**：应用操作时偶尔ANR
**原因**：两个线程以不同顺序获取锁导致死锁
**解决**：统一锁顺序，使用tryLock避免死锁

### 8.3 案例三：大量计算
**现象**：列表滚动时ANR
**原因**：主线程执行复杂计算
**解决**：使用后台线程或预计算优化性能

### 8.4 案例四：定时任务ANR
**现象**：定时任务执行时ANR
**原因**：任务执行时间过长且在主线程执行
**解决**：
- 将任务移至后台线程
- 使用工作队列管理任务
- 优化任务执行逻辑
