# 缺陷文档生成器 (defect-document-generator)

## SKILL技能触发机制

### 触发条件

当用户输入符合以下条件时，系统会自动触发defect-document-generator技能：

1. **总结关键词**：输入中包含"总结"、"生成文档"、"缺陷文档"、"bug文档"等关键词
2. **问题解决标识**：输入中包含"问题已解决"、"已修复"、"解决方案"等标识
3. **文档生成标识**：输入中包含"生成文档"、"创建文档"、"记录"等标识
4. **缺陷信息完整性**：输入中包含问题描述、问题类型、复现概率、复现手法、根本原因、解决方案、验证结果等信息

### 调用方法

#### 直接调用
用户可以直接请求生成文档，系统会自动识别并触发技能：

```
总结以下缺陷并生成文档：
问题描述：应用在点击登录按钮后崩溃
问题类型：崩溃
复现概率：100%
复现手法：1. 打开应用 2. 输入用户名密码 3. 点击登录按钮
根本原因：登录按钮的点击事件处理中存在空指针异常
解决方案：在点击事件处理前添加空值检查
验证结果：问题已解决，不再崩溃
Product Flavor：xxx
```

#### 显式调用
用户也可以通过明确指令调用技能：

```
使用defect-document-generator技能生成以下缺陷的文档：
问题描述：应用在点击登录按钮后崩溃
问题类型：崩溃
复现概率：100%
复现手法：1. 打开应用 2. 输入用户名密码 3. 点击登录按钮
根本原因：登录按钮的点击事件处理中存在空指针异常
解决方案：在点击事件处理前添加空值检查
验证结果：问题已解决，不再崩溃
Product Flavor：xxx
```

### 技能协作

defect-document-generator技能可以与其他技能协作：

1. **与defect-analyzer协作**：当defect-analyzer分析完成后，可以直接触发defect-document-generator生成文档
2. **与SKILL技能协作**：当需要更复杂的技能调用时，通过SKILL技能进行协调

### 响应格式

技能触发后，系统会按照以下格式响应：

1. **生成状态**：确认文档生成状态
2. **文件路径**：提供生成的文档文件路径
3. **文档内容**：展示生成的文档内容摘要
4. **资源结构更新**：确认defect-analyzer的资源结构已更新
5. **下一步建议**：提供后续操作的建议

### 连续触发

支持连续触发机制，当用户需要生成多个缺陷文档时，系统会连续处理每个请求。

## 概述

该技能用于Android应用开发bug或者问题总结md文档生成，当用户处理完缺陷后，自动根据不同问题类型分类创建文件夹，生成标准化的md文档，并动态更新skills\defect-analyzer\SKILL.md技能的文档库。

## 工作流程

1. **接收缺陷信息**：获取用户提供的完整缺陷信息和解决方案
2. **分类处理**：根据问题类型，检查是否存在对应的分类文件夹
3. **创建文件夹**：如果不存在对应分类，创建新的分类文件夹
4. **AI总结提示词**：根据缺陷信息和解决方案，总结出用于提示AI的关键提示词
5. **生成文档**：根据模板生成标准化的缺陷文档，包含AI提示词总结，文档生成前检查下修改的内容，避免AI交互环节遗漏信息
6. **存储文档**：将生成的文档存储到skills\defect-analyzer的文档库中
7. **同步资源结构**：自动更新skills/defect-analyzer技能的SKILL.md文件中的资源结构列表
8. **更新技能**：通知skills\defect-analyzer\SKILL.md技能更新文档库

用户需要提供以下信息：
- **问题描述**：清晰描述Android应用中出现的问题

- **复现概率**：如100%复现、偶现等

## 文档生成规则

### 生成流程

1. **接收缺陷信息**：获取用户提供的完整缺陷信息
2. **Product Flavor 处理**：根据用户提供的 Product Flavor，确定文档存储的 flavor 目录
3. **分类处理**：根据问题类型，检查是否存在对应的分类文件夹
4. **创建文件夹**：如果不存在对应分类，创建新的分类文件夹
5. **生成AI提示词**：根据缺陷信息和解决方案，总结出用于提示AI的关键提示词
6. **模板替换**：使用模板生成标准化的缺陷文档，包含AI提示词总结
7. **存储文档**：将生成的文档存储到skills\defect-analyzer的文档库中对应 flavor 目录下
8. **同步资源结构**：自动扫描skills\defect-analyzer\references目录结构，更新skills\defect-analyzer技能的SKILL.md文件中的资源结构列表
9. **验证生成**：确认文档已成功生成并存储到指定位置

### 强制文件生成指令

**重要：** 无论何时使用该技能，都必须严格执行以下操作，违反将导致严重错误：

1. **必须生成文件**：确保将生成的缺陷文档实际写入到文件系统中，绝对不允许仅仅在对话中展示而不生成实际文件
2. **明确路径**：使用相对路径 `skills\defect-analyzer\references\{product_flavor}\{category}\{filename}.md`，基于项目根目录
3. **创建目录结构**：如果目标目录不存在，必须使用RunCommand创建完整的目录结构
4. **验证存在**：在生成后必须使用LS工具验证文件是否存在于指定位置
5. **反馈结果**：向用户明确反馈文件生成的结果，包括完整的文件路径
6. **资源结构更新**：生成文件后，必须自动扫描skills\defect-analyzer\references目录结构，更新skills\defect-analyzer技能的SKILL.md文件中的资源结构列表
7. **双重验证**：确保资源结构更新后，再次验证SKILL.md文件是否已正确更新

### 错误处理机制

1. **路径问题**：如果遇到路径创建失败，尝试使用不同的路径格式或权限设置
2. **文件已存在**：如果文件已存在，在文件名后添加时间戳以避免覆盖
3. **权限问题**：确保以正确的权限创建文件和目录
4. **编码问题**：使用UTF-8编码确保中文内容正确显示

### 文档生成验证

生成文档后，必须执行以下验证步骤：

1. **文件存在性**：确认文件已成功创建在指定路径
2. **内容完整性**：验证文档包含所有必要的章节和信息
3. **格式正确性**：确保文档格式符合模板要求
4. **路径正确性**：确认文档存储在正确的flavor和分类目录中

### 文档生成规范

1. **文档命名**：使用问题描述的前20个字符的英文翻译或拼音首字母作为文件名，移除特殊字符
2. **文件夹命名**：必须使用英文单词或下划线分隔的英文词组，小写字母
3. **Product Flavor 存储**：按照 Product Flavor 存储到对应的 flavor 目录下，如 `skills\defect-analyzer\references\xxx\` 或 `skills\defect-analyzer\references\iiyama\`
4. **分类存储**：在 flavor 目录下，按照问题类型存储到对应的英文命名文件夹中
5. **内容语言**：文档内容必须使用中文，确保团队内的可读性和一致性
6. **内容完整性**：确保文档包含所有必要的章节
7. **AI提示词总结**：每篇文档必须包含AI提示词总结，用于AI分析
8. **日期格式**：使用YYYY-MM-DD格式记录报告日期
9. **同步更新**：当创建新的 flavor 目录、分类文件夹或文档时，自动更新skills\defect-analyzer技能中的资源结构列表
10. **资源结构同步**：自动扫描skills\defect-analyzer\references目录结构，更新skills\defect-analyzer技能的SKILL.md文件中的资源结构列表，确保文档结构与配置文件保持一致
11. **文件生成限制**：严格禁止生成任何规则外的文件，只允许生成缺陷分析文档到指定的skills\defect-analyzer\references目录中，禁止在项目根目录或其他位置生成临时文件、配置文件或信息文件（如defect_info.txt等）

### 命名转换规则

- **问题类型转文件夹名**：将中文问题类型转换为对应的英文单词或词组
  - 崩溃 → crash
  - 性能问题 → performance
  - UI问题 → ui
  - 网络问题 → network
  - 功能问题 → functionality
  - 兼容性问题 → compatibility

- **文件名生成**：
  - 优先使用问题描述的英文关键词
  - 如无英文关键词，使用拼音首字母组合
  - 避免使用中文和特殊字符
  - 文件名长度不超过20个字符

### 自动分析规则

#### 根本原因自动分析

- **崩溃类型**：
  - 包含"空指针"或"NullPointerException"：代码中存在空指针异常
  - 包含"内存"：应用内存管理不当
  - 包含"ANR"或"无响应"：主线程执行耗时操作
- **UI类型**：
  - 包含"布局"或"显示"：UI布局设计不合理或适配问题
  - 包含"点击"或"交互"：UI交互逻辑存在问题
- **功能类型**：
  - AI圈搜+语音播报+语言：语音播报功能没有正确获取识别结果的语言设置
  - 生成题目+复制：生成题目后的复制功能存在问题

#### 解决方案自动生成

- **崩溃类型**：
  - 空指针异常：添加空值检查
  - 内存问题：优化内存管理
  - ANR：耗时操作移至子线程
- **UI类型**：
  - 布局问题：优化布局设计
  - 交互问题：修复交互逻辑
- **功能类型**：
  - 语音播报语言问题：修改语音播报功能，读取识别结果的语言设置
  - 复制功能问题：检查并修复复制功能
