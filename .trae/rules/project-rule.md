# 项目规则 (Project Rules) for Windows & Android Development

## 基本交互
1.  **对话语言**：请始终使用中文与我对话。
2.  **解释要求**：在回答技术问题或提供代码修改建议时，请先阐述你的计划和理由，并详细解释每一步改动的原因。

## 工作流程
1. 若描述为bug或者问题时，使用技能defect-analyzer
2. 当bug或者问题被解决后并被要求总结时，使用技能defect-document-generator
3. 当需要提交代码时，使用技能gerrit-committer
4. 当用户使用否定词的时候，使用技能skill-optimizer
5. 生成的代码必须严格按照**代码注释规范**进行注释
6.  **加载文件**：第一次项目加载时，如果当前项目的根目录下有claude.md文件，都需要加载当前项目的根目录的claude.md文件。
7.  **项目分析**：在项目加载完成后，如果当前项目的根目录下有claude.md文件，需要分析项目的目录结构和文件依赖关系时优先查看当前项目的根目录的claude.md文件。
8.  **ClAUDE-ERROR.md文件检查**：在项目加载完成后，如果当前项目的根目录下有ClAUDE-ERROR.md文件，需要优先遵循ClAUDE-ERROR.md文件中的建议，避免再次犯同样的错误。

## 系统与代码规范
1.  **操作系统**：我的开发系统是 **Windows**。请在提供任何命令行指令、脚本或文件路径时，都基于 Windows 环境（例如，使用 `.\gradlew` 而不是 `./gradlew`，注意反斜杠和正斜杠的区别）。
2.  **代码注释**：当你生成或修改代码时，请为关键的类、方法、复杂逻辑块添加清晰的中文注释。
3.  **代码质量**：当生成的代码超过20行时，请考虑其模块化和函数的单一职责，避免过长函数。
